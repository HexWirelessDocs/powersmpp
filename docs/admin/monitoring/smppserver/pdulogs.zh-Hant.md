
# 伺服器 PDU 日誌

這個 **SMPP 伺服器 PDU 日誌** 在 iTextPRO 中對 **監測和解決問題** 信件交易 **EMEE( 外部短訊息實體) 使用者** 和 iTextPRO 平臺. 
這些日誌抓取 **下游交通** 並提供有助於有效解決問題的顆粒性細節。

---

## 關鍵特性
- **綜合交易日誌** – 記錄ESME使用者和iTextPRO之間的每封信件交易.
- **解決問題支援** - 對診斷和解決溝通問題至關重要。
- **管理時區** – 日誌顯示在管理員的時區,以便準確參考.

---

## 下游交通
跟蹤 **信件的行程** 從ESMEE使用者到iTextPRO,為交付流程和狀態提供能見度.

## Verbosity 級別

理解 **動詞水平** 協助監測和排除故障:

| **Verbosity 級別** | **說明** | **目的** |
|---------------------|-----------------|-------------|
| **繫結請求** | ESME使用者啟動連線. | 與 iTextPRO 建立連線 。 |
| **繫結響應** | iTextPRO 響應連線請求. | 確認ESME使用者的連線。 |
| **詢問連結請求** 編號 : **回應** | ESME使用者的健康檢查電話和相應的響應. | 驗證iTextPRO會話狀態(建議間隔:30秒). |
| **提交( S) 請求** | ESME使用者啟動訊息請求. | 向 iTextPRO 傳送短訊息 。 |
| **提交 SM 回覆** | iTextPRO回應了信件請求. | 確認短訊息提交。 |
| **交付 SM 請求** | iTextPRO收到的DLR(交付報告),用於傳送資訊。 | 更新提交簡訊的傳送狀態。 |
| **交付 SM 回應** | ESME使用者確認DLR請求. | 確認收到交貨狀態。 |
| **撤銷請求** | ESME使用者啟動未連線請求. | 終止連線會話 。 |

---

## 最佳做法
- **定期審查日誌** - 確保有力監測和迅速發現與交易有關的問題。
- **利用Verbosity Insights(虛擬透視)** - 使用捕獲的PDU細節來保持ESME與iTextPRO之間的有效通訊.
- **即時行動** ——發現異常情況後立即處理,以保持最佳系統性能.
