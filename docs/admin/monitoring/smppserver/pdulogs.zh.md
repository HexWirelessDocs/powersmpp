
# 服务器 PDU 日志

这个 **SMPP 服务器 PDU 日志** 在 iTextPRO 中对 **监测和解决问题** 信件交易 **EMEE( 外部短消息实体) 用户** 和 iTextPRO 平台. 
这些日志抓取 **下游交通** 并提供有助于有效解决问题的颗粒性细节。

---

## 关键特性
- **综合交易日志** – 记录ESME用户和iTextPRO之间的每封信件交易.
- **解决问题支持** - 对诊断和解决沟通问题至关重要。
- **管理时区** – 日志显示在管理员的时区,以便准确参考.

---

跟踪 **信件的行程** 从ESMEE用户到iTextPRO,为交付流程和状态提供能见度.

## Verbosity 级别

理解 **动词水平** 协助监测和排除故障:

| **Verbosity 级别** | **说明** | **目的** |
|---------------------|-----------------|-------------|
| **绑定请求** | ESME用户启动连接. | 与 iTextPRO 建立连接 。 |
| **绑定响应** | iTextPRO 响应连接请求. | 确认ESME用户的连接。 |
| **询问链接请求** 编号 : **回应** | ESME用户的健康检查电话和相应的响应. | 验证iTextPRO会话状态(建议间隔:30秒). |
| **提交( S) 请求** | ESME用户启动消息请求. | 向 iTextPRO 发送短消息 。 |
| **提交 SM 回复** | iTextPRO回应了信件请求. | 确认短消息提交。 |
| **交付 SM 请求** | iTextPRO收到的DLR(交付报告),用于发送信息。 | 更新提交短信的发送状态。 |
| **交付 SM 回应** | ESME用户确认DLR请求. | 确认收到交货状态。 |
| **撤销请求** | ESME用户启动未连接请求. | 终止连接会话 。 |

---

## 最佳做法
- **定期审查日志** - 确保有力监测和迅速发现与交易有关的问题。
- **利用Verbosity Insights(虚拟透视)** - 使用捕获的PDU细节来保持ESME与iTextPRO之间的有效通信.
- **即时行动** ——发现异常情况后立即处理,以保持最佳系统性能.
