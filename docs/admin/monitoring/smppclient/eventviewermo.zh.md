# 事件查看器(MO)——移动源

**导航 :** <span data-ph="0"></span> ➔ <span data-ph="1"></span> ➔ <span data-ph="2"></span>。 。 。 。

## 概览

这个 **事件查看器( MO)** 区域用于监视 **移动原产地(MO)** PowerSMPP内部的信息和相关系统事件. MO信息是网关供应商或终端用户针对该应用程序发出的入境信息。

---

## 目的

事件查看器(MO)帮助管理员监测,审计,并排除系统中所有与MO有关的活动出问题.

!!! info "主要使用案例"
    - 实时监测MO消息活动
    - 检查收到的用户或供应商通信日志
    - 解决与MO有关的交付或路线问题
    - 审查系统产生的MO事件
    - 核查入境流量的网关通信记录

---

## 可用信息

模块为每个日志条目显示以下字段:

| 外地 | 说明 |
|-------|-------------|
| **时间** | MO事件的精确时间戳. |
| **日志类型** | 活动的分类—— <span data-ph="0"></span> 或者说 <span data-ph="1"></span>。 。 。 。 |
| **消息** | MO事件或活动的描述. |

## 日志类型

| 日志类型 | 说明 |
|----------|-------------|
| **资讯** | 信息系统事件,成功网关更改,用户列表更新. |
| **错误** | 显示失败的错误级别事件,拒绝MO消息,或系统例外. |

---

## 特征

!!! info "事件查看器( MO) 能力"
    - 实时查看MO事件日志.
    - 监测网关和用户入境活动。
    - 跟踪收到的MO信件请求 。
    - 解决与MO有关的问题。
    - 使用 **刷新** 按钮。
    - 过滤日志 **日志类型** (信息/ 错误).
    - 过滤日志 **日期范围** 历史回顾。

---

## 日期范围过滤器

管理员可以选择一个特定的日期区域进行检索 **历史MO事件日志**这有助于追溯性解决问题、合规审计和业务审查。

!!! tip
 当追踪间歇性问题时,设定 **日志类型 = 出错** 首先只注重失败,然后扩大至 **全体** 如果窗口中看不到出错事件。
