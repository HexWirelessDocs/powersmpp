# 監測

iTextPRO 提供全面 **監測工具** 以確保最佳的簡訊傳送效能和系統穩定性。 這包括: **實時現場監測** 對交通的洞察力和強大的 **PDU 標記器** 深層資訊分析。

---

## 現場監測

這個 **現場監測** 模組在 iTextPRO 動態跟蹤和分析與短訊息流量相關的關鍵資料點,啟用 **實時決策** 用於路由和效能最佳化。

### 主要惠益
- **主動交通管理** ——根據活資料動態管理簡訊流量.
- **最佳化路線** - 作出知情的路線決定來提高交付率。
- **高效的資源分配** - 在高峰時段戰略性地分配資源。
- **增強業績** ——以實時見識提高反應能力和吞吐量.

**總結**,現場監測確保使用者 **隨時可見** 進入短訊息流量,特別是在高需求時期.

---

## PDU 日誌

iTextPRO 使用一個 **PDU (協議資料單位) 記錄器** 以獲取和記錄每條進出簡訊。 這個工具對 **排除出問題**, (中文(簡體) ). **監測**,以及 **維持** 系統健康。

### 關鍵特性
- **實時信件行程** - 實時記錄每條資訊,以便立即進行分析。
- **過濾能力** – 點選一鍵追蹤訊息的行程 。
- **PDU 型別支援** – 檢查StoppetSM,交付SM,平德,Unbind等.
- **可見和保留** - 日誌跟隨 **管理時區** 並保留用於 **3天時間**。 。 。 。
- **上游交通檢查** – 透過從下拉列表中選擇閘道器來檢視訊息流.
- **解決問題支援** - 快速診斷分娩或SMPP會話問題。

![PDU Logs](images/pdulogs1.png)

### 使用準則
1. 進入 **PDU 標記器** 從 iTextPRO 介面。
2. 應用 **過濾器** 以隔離和檢查特定資訊。
3. 使用 **上游交通記錄** 以驗證信件行程。
4. 執行 **定期監測** 保持系統可靠性。

## PDU 日誌中的Verbosity 級別

iTextPRO 的 PDU 記錄支援多個 **動詞水平**,提供了對iTextPRO和SMPP閘道器之間通訊的詳細見解.

| Verbosity 級別 | 目的 | 行動 |
|-----------------|---------|--------|
| **繫結請求** | 啟動 SMPP 繫結 | iTextPRO 傳送連線 SMPP 閘道器的請求 |
| **繫結響應** | 確認 SMPP 繫結 | SMPP 閘道器響應繫結請求 |
| **詢問連結請求/答覆** | SMPP會議的健康檢查 | iTextPRO 每30s傳送請求;閘道器響應 |
| **提交( S) 請求** | 信件提交請求 | iTextPRO 向 SMPP 閘道器傳送短訊息 |
| **提交 SM 回覆** | 提交確認 | 閘道器響應信件提交 |
| **交付 SM 請求** | 交付報告(DLR)接收 | SMPP 閘道器傳送送出狀態 |
| **交付 SM 回應** | 承認德國航天中心 | iTextPRO 確認收到 DLR |
| **撤銷請求** | 會話終止 | iTextPRO 或閘道器啟動未連線的請求 |

**重要性 :** 這些日誌給管理員一個 **訊息流的顆粒檢視**幫助有效發現、診斷和解決問題。
