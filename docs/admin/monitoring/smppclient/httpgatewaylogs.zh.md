# HTTP 网关日志

**导航 :** <span data-ph="0"></span> ➔ <span data-ph="1"></span> ➔ <span data-ph="2"></span>。 。 。 。

## 概览

这个 **HTTP 网关日志** 区域允许管理员查看完整的 HTTP **请求** 财务报告和审定财务报表 **回应** PowerSMPP应用程序与已配置的网关供应商之间交流. 这些记录对于诊断交付失败、核实API有效载荷和审计网关互动至关重要。

---

## 查看日志的步骤

1. 选择 **HTTP 网关** 从 **网关名称** 放下枪
2. 选择需要的 **日期范围** 使用取枣机。
3. 选择 **日志类型** (单位:千美元)<span data-ph="0"></span> 编号 : <span data-ph="1"></span> 编号 : <span data-ph="2"></span>视需要而定。
4. 选择 **线性** 如果需要进行颗粒过滤,则级别。
5. 点击 **提交** 以获取并显示匹配日志。

## 可用的日志区域

### 请求正文

这个 **请求正文** 包含从PowerSMPP应用程序传输到网关供应商的完整有效载荷。

![HTTP Gateway Logs — Request Body expanded](images/httplogs-02-request-body.png)

!!! info "请求正文 - 包含数据"
    - **移动号码** ——短信的目的地编号
    - **发件人标识** – 使用的原地址/发件人ID
    - **请求参数** ——发送到网关的完整API参数
    - **主机/ IP 细节** ——网关端点的IP地址和端口
    - **提交时间** - 发出请求的时间戳

### 反应机构

这个 **反应机构** 包含网关供应商提供的确认和状态数据。

![HTTP Gateway Logs — Response Body expanded](images/httplogs-03-response-body.png)

!!! info "反应机构——包括数据"
    - **网关响应** - 网关返回的原始反应
    - **状况信息** - 交付或接受状态代码
    - **错误细节** — 错误代码和失败的提交描述
    - **鸣谢** ——确认网关处理请求.

## 过滤选项

管理员可以使用下列附加过滤器来改进日志视图:

| 过滤器 | 使用 |
|--------|-----|
| **IP 地址** | 通过网关服务器 IP 过滤日志 。 |
| **发件人标识** | 通过发件人ID过滤日志. |
| **移动号码** | 按目的地移动编号过滤日志 。 |

## 典型的解决问题

1. 一场运动报道出乎意料的失败或未传达的信息.
2. 打开 **HTTP 网关日志**,选择受影响的网关和涵盖该问题的日期范围。
3. 设定 **日志类型** 改为 <span data-ph="0"></span> 到表面只显示失败的交换。
4. 扩展 **请求正文** 证实外出有效载荷是正确的。
5. 扩展 **反应机构** 读取供应商返回的实际错误代码/描述。
6. 使用 **IP 地址**, (中文(简体) ). **发件人标识**,或 **移动号码** 过滤器,为供应商的支助小组隔离特定测试样本。
