# 事件檢視器(MO)——移動源

**導航 :** <span data-ph="0"></span> ➔ <span data-ph="1"></span> ➔ <span data-ph="2"></span>。 。 。 。

## 概覽

這個 **事件檢視器( MO)** 區域用於監視 **移動原產地(MO)** PowerSMPP內部的資訊和相關係統事件. MO資訊是閘道器供應商或終端使用者針對該應用程式發出的入境資訊。

---

## 目的

事件檢視器(MO)幫助管理員監測,審計,並排除系統中所有與MO有關的活動出問題.

!!! info "主要使用案例"
    - 實時監測MO訊息活動
    - 檢查收到的使用者或供應商通訊日誌
    - 解決與MO有關的交付或路線問題
    - 審查系統產生的MO事件
    - 核查入境流量的閘道器通訊記錄

---

## 可用資訊

模組為每個日誌條目顯示以下欄位:

| 外地 | 說明 |
|-------|-------------|
| **時間** | MO事件的精確時間戳. |
| **日誌型別** | 活動的分類—— <span data-ph="0"></span> 或者說 <span data-ph="1"></span>。 。 。 。 |
| **訊息** | MO事件或活動的描述. |

## 日誌型別

| 日誌型別 | 說明 |
|----------|-------------|
| **資訊** | 資訊系統事件,成功閘道器更改,使用者列表更新. |
| **錯誤** | 顯示失敗的錯誤級別事件,拒絕MO訊息,或系統例外. |

---

## 特徵

!!! info "事件檢視器( MO) 能力"
    - 實時檢視MO事件日誌.
    - 監測閘道器和使用者入境活動。
    - 跟蹤收到的MO信件請求 。
    - 解決與MO有關的問題。
    - 使用 **重新整理** 按鈕。
    - 過濾日誌 **日誌型別** (資訊/ 錯誤).
    - 過濾日誌 **日期範圍** 歷史回顧。

---

## 日期範圍過濾器

管理員可以選擇一個特定的日期區域進行檢索 **歷史MO事件日誌**這有助於追溯性解決問題、合規審計和業務審查。

!!! tip
 當追蹤間歇性問題時,設定 **日誌型別 = 出錯** 首先只注重失敗,然後擴大至 **全體** 如果視窗中看不到出錯事件。
