
# 信件正文正常化

這個 **信件正文正常化** 規則是一個內在特徵 **電源 SMPP**,旨在賦予管理員在提交閘道器前修改和完善訊息內容的靈活性. 
這確保所有發出的資訊格式正確,提高可讀性和有效性。

這份檔案解釋瞭如何 **信件正文正常化** 工作,以及管理員如何可以配置它,根據預先定義的規則自動調整訊息內容——確保一致性和遵守閘道器要求.

---

## 關鍵特性

信件正態化規則擴充套件現有的 **OA/DA 正常化規則**。它提供了多種操作訊息內容的方法,包括:

- **守則/OTP的提取:** 從電文中自動檢測和提取OTP或特定程式碼. 
- **新增字首或字尾 :** 在訊息之前或之後插入特定文字以保持標準格式。 
- **文字替換 :** 按照預先定義的規則替換某些單詞或短語,以達到更好的一致性.

這些選項幫助管理員確保所有資訊符合閘道器標準,改善總體資訊傳遞體驗.

## 規則配置

一旦點選 **新增新內容**,以多個欄位顯示配置頁面。

### **規則名稱**
為規則定義一個描述性和有意義的名稱。

---

### **條件設定**

#### 1. 選擇介面
指定規則適用的介面。 您可以選擇以下一個或多個:

- 韋伯 
- API 密碼 
- SMPP 軟體 
- 所有介面 

![Interface Selection](images/mbn3.png)

這使得根據操作介面可以有針對性地應用規則.

---

#### 2. 使用者
選擇規則是否適用於:
- 具體 **使用者**,或 
- **沒有** (適用於所有使用者).

---

#### 3. 發件人身份證
基於特定匹配模式配置發件人標識條件 :

- **平等** - 完全匹配指定的發件人ID 
- **開始方式** — 如果發件人ID以特定關鍵字開頭的觸發器 
- **結束方式** - 如果發件人ID以特定關鍵詞結尾,則應用 
- **包含** - 如果關鍵詞在發件人ID的任何地方找到,則應用 

![Sender ID Condition](images/mbn4.png)

---

#### 4. 目的地地址
類似職能 **發件人標識**,允許同樣的條件—— *平等*, (中文(簡體) ). *開始方式*, (中文(簡體) ). *結束方式*, (中文(簡體) ). *包含* ——應用基於目的地地址格式的規則.

---

#### 5. 電文
與上述相同的有條件選項可以被應用於訊息內容,從而能夠精確地進行格式化控制.

- 這保證了訊息內容的最佳化和格式化,防止規則衝突.

---

## 內容提要

**信件正文正常化** 在Power SMPP中授權管理員:
- 抽取檢察官辦公室或程式碼, 
- 新增字首/字尾, 
- 替換詞語或短語, 
- 結合多種規則, 
- 每個使用者、傳送者或介面應用規則。 

這一功能確保所有資訊保持統一的格式並遵守閘道器要求——提高資訊傳送的可靠性和專業性.
