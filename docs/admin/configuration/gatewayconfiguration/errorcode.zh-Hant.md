
## 網關出錯程式碼

![Gateway Error Codes](images/error1.png)

**網關出錯程式碼** 在 iTextPRO 中允許您解釋和管理來自 **簡訊息服務中心( SMSC)** 當信件傳送失敗時。 這一功能提高了交付失敗的能見度並使得基於特定出錯程式碼的信用回滾成為可能.

---

![Configure Error Codes](images/error2.png)

### 1. 概覽
當簡訊中心無法傳送訊息時, 它返回 a **非零出錯程式碼** 帶有參考訊息狀態。 這些稱為: **網關出錯程式碼**。 。 。 。

### 2. 目的
iTextPRO使管理員能夠配置和對映這些出錯程式碼. 當a **交付報告(DLR)** 收到後,iTextPRO檢查配置並:
- 顯示 a **自定義錯誤標籤**
- 地圖到一個 **標準 SMPP 狀態** 
以上資訊均列於 **管理員** 財務報告和審定財務報表 **使用者報告**提高透明度。

### 3. 先決條件
在配置前,獲取一個 **網關出錯程式碼列表** 從你的簡訊中心。

---

### 4. 配置步驟

#### a. 選擇門戶
選擇您正在配置錯誤程式碼的閘道器 。

#### b. 錯誤程式碼
輸入 **特定錯誤程式碼** 從您的簡訊中心收到。

#### c. 顯示出錯標記
插入 **參考狀態** 或標記名稱來自短訊息中心(例如, <span data-ph="0"></span>, (中文(簡體) ). <span data-ph="1"></span>) (中文(簡體) ).

#### d. 標準出錯程式碼(已安裝)
將錯誤對映到標準的 SMPP 狀態之一 :
- <span data-ph="0"></span>
- <span data-ph="0"></span>
- <span data-ph="0"></span>
- <span data-ph="0"></span>
- <span data-ph="0"></span>

#### e. 錯誤描述
提供a **簡介** 錯誤標記。 這將顯示在 **使用者報告** 幫助瞭解交付狀況。

#### f. 活動
切換錯誤程式碼 **開啟或關閉** 視需要而定。

#### g. 信貸回滾
啟用此選項 **收回使用者信用** 如果信件與映射出的錯誤程式碼相悖,則自動失敗 。

---

### 5. 散裝上傳

- 使用 **批次上傳** 特性以同時配置多個錯誤程式碼。
- 點選 **"煙花上傳"**,則:
  - **下載樣本檔案**
  - **地圖列標題** 在匯入嚮導中

---

### 6. 提交
配置後,單擊 **"投誠"** 以儲存選定閘道器的錯誤程式碼。

📌 **說明:** 
正確的錯誤程式碼配置改進了監測,並允許精確跟蹤 **訊息傳送失敗**, (中文(簡體) ). **回滾動作**,以及 **報告準確性**。 。 。 。
