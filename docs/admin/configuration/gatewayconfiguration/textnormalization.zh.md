
# 信件正文正常化

这个 **信件正文正常化** 规则是一个内在特征 **电源 SMPP**,旨在赋予管理员在提交网关前修改和完善消息内容的灵活性. 
这确保所有发出的信息格式正确,提高可读性和有效性。

这份文件解释了如何 **信件正文正常化** 工作,以及管理员如何可以配置它,根据预先定义的规则自动调整消息内容——确保一致性和遵守网关要求.

---

## 关键特性

信件正态化规则扩展现有的 **OA/DA 正常化规则**。它提供了多种操作消息内容的方法,包括:

- **守则/OTP的提取:** 从电文中自动检测和提取OTP或特定代码. 
- **添加前缀或后缀 :** 在消息之前或之后插入特定文本以保持标准格式。 
- **文本替换 :** 按照预先定义的规则替换某些单词或短语,以达到更好的一致性.

这些选项帮助管理员确保所有信息符合网关标准,改善总体信息传递体验.

## QQ 存取信件正态化

配置信件正文正常化 :

1. 导航到 **配置模块**。 。 。 。 
2. 选择 **网关配置**。 。 。 。 
3. 选择 **信件正文正常化**。 。 。 。 
4. 点击 **添加新内容** 以建立新的正常化规则。

## 规则配置

一旦点击 **添加新内容**,以多个字段显示配置页面。

### **规则名称**
为规则定义一个描述性和有意义的名称。

---

### **条件设置**

#### 1. 选择接口
指定规则适用的接口。 您可以选择以下一个或多个:

- 韦伯 
- API 密码 
- SMPP 软件 
- 所有接口 

![Interface Selection](images/mbn3.png)

这使得根据操作界面可以有针对性地应用规则.

---

#### 2. 用户
选择规则是否适用于:
- 具体 **用户**,或 
- **没有** (适用于所有用户).

---

#### 3. 发件人身份证
基于特定匹配模式配置发件人标识条件 :

- **平等** - 完全匹配指定的发件人ID 
- **开始方式** — 如果发件人ID以特定关键字开头的触发器 
- **结束方式** - 如果发件人ID以特定关键词结尾,则应用 
- **包含** - 如果关键词在发件人ID的任何地方找到,则应用 

![Sender ID Condition](images/mbn4.png)

---

#### 4. 目的地地址
类似职能 **发件人标识**,允许同样的条件—— *平等*, (中文(简体) ). *开始方式*, (中文(简体) ). *结束方式*, (中文(简体) ). *包含* ——应用基于目的地地址格式的规则.

---

#### 5. 电文
与上述相同的有条件选项可以被应用于消息内容,从而能够精确地进行格式化控制.

## 优先权和相容性

- **信件正文正常化** 执行 **在此之前** 页:1 **OA/DA 正常化**。 。 。 。 
- 这保证了消息内容的优化和格式化,防止规则冲突. 
- 两项正常化规则都分层进行,以便准确提交网关。

---

## 内容提要

**信件正文正常化** 在Power SMPP中授权管理员:
- 抽取检察官办公室或代码, 
- 添加前缀/后缀, 
- 替换词语或短语, 
- 结合多种规则, 
- 每个用户、发送者或接口应用规则。 

这一功能确保所有信息保持统一的格式并遵守网关要求——提高信息发送的可靠性和专业性.
