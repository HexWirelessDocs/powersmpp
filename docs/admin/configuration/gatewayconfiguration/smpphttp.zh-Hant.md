# 配置

## SMPP 閘道器(MO/MT)

這個 **iTextPRO 應用程式** 優先順序a **方便使用者的經驗** 透過靈活的設計介面和簡化的配置工作流程. 目標是擺脫CLI的複雜性,使應用程式所有者沉浸在 **圖形使用者介面( GUI)**- 基於環境。 綜合 **通訊引擎** 處理所有後端命令,精簡您的操作任務。

---

![Manage Gateway](images/httpgateway1.png)

---

這個 **"管理閘道器"** 特性授權使用者處理與外部的連線 **簡訊息服務中心** 透過 **SMPP 軟體** 財務報告和審定財務報表 **HTTP 密碼** 協議。

對於 **SMPP 軟體**頁:1 **單綁** 允許移動式灌溉(MO)、移動終端(MT)和交付報告(DLR)操作。 iTextPRO 支援 **多個 SMPP 閘道器**,允許 **冗餘** 財務報告和審定財務報表 **成本效益高的路線**。 。 。 。

### 配置新閘道器

要設定 SMPP 聯結器 :

1. 點選 **“新增新”**。 。 。 。
2. 輸入您提供的證書 **閘道器供應商** 或者說 **電信運營商**。 。 。 。

---

![SMPP Gateway Credentials](images/httpgateway2.png)

---

#### 所需全權證書

| 外地 | 說明 |
|-------|-------------|
| **閘道器名稱** | 一個方便使用者的名稱來識別您的閘道器 |
| **IP 地址** | 您的短訊息/ 供應商的IP |
| **系統標識** | 您的供應商/ SMSC 提供的使用者名稱 |
| **密碼** | 用於對 SMSC 的認證 |
| **Tx埠/ Rx埠/ TxR埠** | 傳送器、接收器和收發器連線埠 |
| **系統型別** | *(備選)* 僅在供應商提出要求時輸入 |

!!! warning
 雙向檢查根據短訊息/供應商文件的所有值,以確保連線成功。

### QQ 閘道器屬性

![Gateway Properties](images/httpgateway3.png)

配置 SMPP iTextPRO中用於最佳效能的閘道器屬性 :

1. **保持生命( 秒) :** 
 間隔 *詢問連結* 讓會議保持活力

2. **閘道器開啟時間/ 關閉時間 :** 
 界定經常用於遵守的工作時間 **不要煩惱** 政策。

3. **閘道器編碼 :** 
 字符集選擇與telco/SMSC相容.

4. **轉換信件ID :** 
 允許在 **十進位制 QX 十六進位制** 用於精確 DLR 的訊息ID格式 。

5. **時區:** 
 所有報告都將反映這一選定的時區。

6. **停止損失價格 :** 
 設定 **允許的最大閘道器成本** 當使用盲道。

7. **每秒通量( TPS) :** 
 根據供應商的能力界定。

### TON/NPI 設定

![TON/NPI Setup](images/httpgateway4.png)

- **TON(數字型別):** 
 根據短報文件選擇(例如國際、字母等)

- **NPI(數字計劃指標):** 
 表示使用的編號標準(E.164、ISDN等)

- **會話設定 :** 
 每個供應商分配配置 Tx, Rx,和 TxR 會話 。

---

### 角色執行( R)

![Gateway Roles](images/httpgateway5.png)

- **活動 :** 
 將閘道器標為直播通道 並準備通路

- **是預設 :** 
 只有一條閘道器可以被標記為預設. 不匹配路徑的信件到這裡 。

- **是 Async 嗎 :** 
 啟用 **同步模式** 以更快的速度提交信件。

- **盲從:** 
 允許向國家提交信件 **未確定成本價格**。 。 。 。

!!! tip
 配置後,單擊 **"救贖"** 傳送一個 **飛行繫結請求** ——不需要手動重啟.
