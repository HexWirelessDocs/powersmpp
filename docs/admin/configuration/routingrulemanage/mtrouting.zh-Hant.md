## 執行規則管理器

騎馬在保持競爭優勢和使收入最大化方面發揮著關鍵作用。 這個 **執行規則管理器** 在iTextPRO中,能夠透過動態和智慧的路由邏輯,高效地將使用者簡訊流量轉發到最終目的地.

SMPP應用程式支援多條路由路徑. iTextPRO簡化了這種複雜性,允許您建立動態的MT(Mobile Entermined)路由規則來提升交付效能,降低成本,並智慧地分配流量.

主要好處包括:
- 根據交付時間選擇閘道器
- 透過成本最低的路由最佳化成本
- HTTP 和 SMPP 協議之間的動態拖動
- 實時交通分配和負荷平衡

一旦進入 **MT 規則** 中,將顯示已配置的路由規則。 你們可以 **編輯** 任何規則,請點選編輯圖示。

!!! tip
 不需要手動重啟或重新裝入 。 所有路由規則的更新在"一飛"上被應用.

### 管理 MT 執行規則

![Manage Routing Rule](images/routingrule1.png)

要建立新規則,請單擊 **"增加"新MT規則".** 按鈕。 您將被要求配置以下內容:

#### 1. 聯合國 **規則名稱**
- 輸入一個友好、描述性的名稱,以便於識別。

#### 2. 聯合國 **過濾型別**
- 有兩個過濾器型別:

##### **透過過濾器**
- 為全球通路政策所設計.
- 建議建立高優先穿行規則作為倒行道.

![Pass Through Filter](images/routingrule2.png)

##### **自定義過濾器**
- 基於更具體引數的路由資訊 :

![Custom Filter](images/routingrule3.png)

- **國家程式碼**:選擇國家進行線路短訊息流量.
- **管理協委會/多國**:選擇選定國家的特定行動網路.
- **使用者**: 將規則應用於個人使用者。
- **使用者組**: 將規則應用於特定的使用者群.
- **開始日期和結束日期**: 設定規則的有效期.
- **源地址**: 定義源地址–特定路由.
- **目標地址**: 定義目的地地址 - 特定路線.

### 規則政策

您可以根據業務要求或SLA來定義路線政策. 現有路線政策包括:

#### **LCR( 成本最低的路線)**
- 透過供應商閘道器的交通路線,為某一目的地提供最低的配置成本。
- 有助於最佳化定價並增加利潤幅度。

#### **固定閘道器**
- 所有流量都透過一個單一的,預設的閘道器進行通路.

#### **輪迴羅賓**
- 將流量平均分佈在選定的閘道器之間。
- 確保所有配置的閘道器得到最佳使用。

#### **已分發**
- 高階負載平衡方法.
- 根據百分率(如60%、30%、10%)向多個閘道器的線路流量。

#### **ACD/DLR(公認交付)**
- 也被稱為以快遞為基礎的出行路線.
- 向交貨比率最高的供應商閘道器(DLR)運輸,確保實時質量。

### 優先處理

!!! info "優先處理"
 如果多條路由規則匹配一條訊息, **iTextPRO 將選擇優先順序值最高的規則** (數字最高).

這種強大的路由邏輯保證了您的簡訊流量能夠高效,經濟,並符合業務邏輯——不需要服務中斷或系統重啟.
