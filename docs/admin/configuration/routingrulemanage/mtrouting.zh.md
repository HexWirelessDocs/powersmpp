## 运行规则管理器

骑马在保持竞争优势和使收入最大化方面发挥着关键作用。 这个 **运行规则管理器** 在iTextPRO中,能够通过动态和智能的路由逻辑,高效地将用户短信流量转发到最终目的地.

SMPP应用程序支持多条路由路径. iTextPRO简化了这种复杂性,允许您创建动态的MT(Mobile Entermined)路由规则来提升交付性能,降低成本,并智能地分配流量.

主要好处包括:
- 根据交付时间选择网关
- 通过成本最低的路由优化成本
- HTTP 和 SMPP 协议之间的动态拖动
- 实时交通分配和负荷平衡

一旦进入 **MT 规则** 中,将显示已配置的路由规则。 你们可以 **编辑** 任何规则,请点击编辑图标。

!!! tip
 不需要手动重启或重新装入 。 所有路由规则的更新在"一飞"上被应用.

### 管理 MT 运行规则

![Manage Routing Rule](images/routingrule1.png)

要创建新规则,请单击 **"增加"新MT规则".** 按钮。 您将被要求配置以下内容:

#### 1. 联合国 **规则名称**
- 输入一个友好、描述性的名称,以便于识别。

#### 2. 联合国 **过滤类型**
- 有两个过滤器类型:

##### **通过过滤器**
- 为全球通路政策所设计.
- 建议建立高优先穿行规则作为倒行道.

![Pass Through Filter](images/routingrule2.png)

##### **自定义过滤器**
- 基于更具体参数的路由信息 :

![Custom Filter](images/routingrule3.png)

- **国家代码**:选择国家进行线路短消息流量.
- **管理协委会/多国**:选择选定国家的特定移动网络.
- **用户**: 将规则应用于个人用户。
- **用户组**: 将规则应用于特定的用户群.
- **开始日期和结束日期**: 设定规则的有效期.
- **源地址**: 定义源地址–特定路由.
- **目标地址**: 定义目的地地址 - 特定路线.

### 规则政策

您可以根据业务要求或SLA来定义路线政策. 现有路线政策包括:

#### **LCR( 成本最低的路线)**
- 通过供应商网关的交通路线,为某一目的地提供最低的配置成本。
- 有助于优化定价并增加利润幅度。

#### **固定网关**
- 所有流量都通过一个单一的,预设的网关进行通路.

#### **轮回罗宾**
- 将流量平均分布在选定的网关之间。
- 确保所有配置的网关得到最佳使用。

#### **已分发**
- 高级负载平衡方法.
- 根据百分率(如60%、30%、10%)向多个网关的线路流量。

#### **ACD/DLR(公认交付)**
- 也被称为以快递为基础的出行路线.
- 向交货比率最高的供应商网关(DLR)运输,确保实时质量。

### 优先处理

!!! info "优先处理"
 如果多条路由规则匹配一条消息, **iTextPRO 将选择优先级值最高的规则** (数字最高).

这种强大的路由逻辑保证了您的短信流量能够高效,经济,并符合业务逻辑——不需要服务中断或系统重启.
