# OAuth 处理器配置

单位 **HTTP 网关**,我们支持 **3种授权**数字 :

| {\fn方正粗倩简体\fs12\an8\1cHFFFF00\b0} | 类型 | 说明 |
|---|------|-------------|
| 1个 | **无授权** | 无需批准。 |
| 2个 | **基本认证** | API的安全认证需要用户名和密码. |
| 3个 | **OAuth 2.0 数据** | 最新版本的授权,用于在一段时间后重新生成新的证书,以保持API的高安全性。 **OAuth 处理器** API (英语). |

在这份文件中,我们将解释为执行《公约》所需的所有步骤和资料。 **OAuth 处理器** HTTP 网关的配置 。

## OAuth 处理器字段

| 外地 | 需求 | 说明 |
|-------|----------|-------------|
| **名称** | 对 | OAuth 处理器的用户友好名称 。 它有助于在应用程序中方便地识别和管理不同的OAuth处理器. |
| **调试 URL** | 对 | 应用程序将请求 OAuth 令牌的 URL 端点 。 是销售商提供的URL来获取访问令牌. |
| **过期时间** | 对 | 出入证有效期为几分钟。 这一时期后,信使会到期,需要产生新的信使. |
| **托肯过期代码** | 对 | 表示该令牌已过期的错误代码 。 当收到这个错误代码时,系统会知道它需要刷新符号. |
| **请求方法** | 对 | HTTP 方法用于从 Token URL 中请求令牌—— <span data-ph="0"></span> 或者说 <span data-ph="1"></span>。 。 。 。 |
| **反应类型** | 对 | 从Token URL收到回复的格式—— <span data-ph="0"></span>, (中文(简体) ). <span data-ph="1"></span>,或 <span data-ph="2"></span>。 。 。 。 |
| **访问托肯字段** | 可选 | 响应中包含访问符的字段名称 。 系统会从这个字段取出访问符来认证未来的请求. |
| **刷新托肯字段** | 可选 | 包含刷新符号的响应中的字段名称 。 刷新令牌用于在当前令牌到期时获得新的访问令牌. 这个领域是可选的,取决于供应商的执行情况。 |

## 如何运作

1. 当需要通过 HTTP 网关发送信件时 **OAuth 2.0 数据**, Power SMPP 首先检查一个有效的(非过期)访问符是否已经缓存.
2. 如果存在有效标志,它会附在外向API呼叫上(通常通过 <span data-ph="0"></span> 头曰.
3. 如果不存在有效符号——或者符号已过期并配置 **托肯过期代码** 由网关返回 。 **调试 URL** 带有配置的 <span data-ph="0"></span>, (中文(简体) ). <span data-ph="1"></span>,以及 <span data-ph="2"></span> 双人
4. 使用 **反应类型**,则 **访问托肯字段** 中提取出,并在 **过期时间**。 。 。 。
5. 出境消息请求现在使用新获得的令牌,直到它再次到期.

## 将 OAuth 处理器链接到 HTTP 网关

在保存 OAuth 处理器后 :

1. 打开您想要与 OAuth 安全的 HTTP 网关 。
2. 下 **第1节:必要的全权证书**设置 **认证** 改为 **OAuth 2.0 数据**。 。 。 。
3. 从 **OAuth 处理器** 放下,选择你刚创建的处理器。
4. 救出门户.

HTTP Gateway 现在将使用所配置的OAuth Handler来自动获取并刷新令牌——不需要手动令牌旋转.

!!! tip
 留着 **过期时间** 略低于供应商公布的价值(例如,设定) <span data-ph="0"></span> 如果卖家的证物是最后的 <span data-ph="1"></span> 分钟). 这避免了到期后第一次请求在刷新被触发前失败的比赛条件.
