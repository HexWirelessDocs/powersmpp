# HTTP MO 处理器

## 概览

这个 **HTTP MO 处理器** 在 iTextPro 中用于接收和处理收到的 **MO( 移动原生)** 电信运营商或网关供应商发来的信息。 处理器充当API端点,接受MO请求,将收到的数据转发到平台,以进一步进行路由和处理.

HTTP MO 处理器配置定义 :

- 交流方法
- 频道细节
- 有效载荷映射
- 安全限制
- 端点生成

!!! warning "先决条件"
 此配置为 **强制** 在创建MO Routing规则之前.

## 处理器配置参数

在创建 HTTP MO Handler 时必须配置以下参数 。

### 1] 处理器名称

这个 **处理器名称** 用于在平台内唯一识别MO Handler配置。

此名称在内部用于:

- 运行配置
- 处理器选择
- MO 交通制图
- 行政和解决问题

!!! example
    ```
    MO_HANDLER_INDIA_01
    ```

---

### 2] 频道类型

这个 **频道类型** 定义与运入的MO流量相关的电信号码类型。

**支持的频道类型 :**

| 类型 | 说明 |
|------|-------------|
| **长码** | 用于双向消息通信的标准手机号码. |
| **短码** | 一般用于企业竞选,投票系统,订阅,或促销服务的短数字代码. |

所选择的频道类型应匹配操作员或供应商配置.

![Manage MO Handler — full configuration form](images/mohandler-02-add-new.png)

---

### 3] 频道号码

这个 **频道号码** 代表收到MO流量的实际目的地号码。

!!! example
    ```
    567890
    ```

此数字必须配置 **完全按照规定** 由电信运营商或网关供应商提供。

---

### 4] 当地终点

这个 **本地终点** 由 iTextPro 创建处理器配置后自动生成。

此端点作为接收 HTTP MO 请求的接收 URL 。

生成的端点 URL 通常与 :

- 短消息网关供应商
- 电信运营商
- 汇总数
- 第三方平台

供应商利用这个端点将收到的MO消息推入iTextPro平台.

**示例流程:**

```
Operator/Vendor → Local Endpoint → iTextPro Processing
```

---

### 5] 白名单IP

这个 **白名单 IP** 区域用于通过限制访问仅限授权的IP地址来保障MO端点的安全.

只有从配置的IP地址收到的请求才会被平台接受.

**目的:**

- 防止未经许可进入
- 加强API安全
- 限制未知流量
- 保护MO端点不被滥用

!!! example
    ```
    192.168.10.20
    ```

!!! tip
 可以根据操作要求配置多个IP.

---

### 6] 方法

这个 **方法** 定义供应商在发送MO请求时使用的HTTP通信方法.

**支持的方法:** <span data-ph="0"></span>, (中文(简体) ). <span data-ph="1"></span>。 。 。 。

#### 获取方法

在 GET 方法中:

- 参数在URL内传输.
- 数据作为查询参数发送.
- 适合轻量级融合.

!!! example
    ```
    https://domain.com/mo?originator=9876543210&destination=567890&message=TEST
    ```

#### POST 方法

在 POST 方法中:

- 参数在HTTP请求机构内传递.
- 支持结构化和更大的有效载荷。
- 常用于现代API集成.

**福利:**

- 加强安保
- 更清洁的请求结构
- 支持 JSON / XML 有效载荷
- 适用于复杂的整合

## 有效载荷配置

这个 **有效载荷** 区域定义了将如何在 iTextPro 内映射收到的请求参数。

正确的有效载荷映射是 **强制** 用于成功的MO处理. 配置有效载荷参数,具体如下:

| 平台参数 | 供应商参数 |
|--------------------|------------------|
| **发件人** | <span data-ph="0"></span> |
| **目标** | <span data-ph="0"></span> |
| **消息** | <span data-ph="0"></span> |

### 有效载荷参数描述

#### 发件人

这个 **发件人** 参数代表收到MO消息的发件人移动号码.

!!! example
    ```
    9876543210
    ```

#### 目标

这个 **目标** 参数代表 **短码** 或者说 **长码** MO 消息发送的编号。

!!! example
    ```
    567890
    ```

#### 消息

这个 **消息** 参数代表最终用户提交的实际文本内容。

!!! example
    ```
    ASKRK BALANCE
    ```

## 保存处理器配置

完成所有配置后 :

1. 校验所有配置的细节 。
2. 点击 **保存**。 。 。 。

HTTP MO Handler现已成功配置并准备接收MO流量.

!!! tip "下一个步骤"
 处理器保存后,立即创建 **MO 运行规则** 以定义进取的MO流量将如何被过滤并交付给终端用户。
