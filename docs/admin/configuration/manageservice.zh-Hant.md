
## 管理服務

這個 **管理服務** iTextPRO 中的段落為各種區域提供可見度 **背景和前景服務** 驅動平臺的核心功能 這個介面主要面向專家使用者或管理員.

!!! danger "謹慎"
 雖然從網路介面可以停止或啟動服務,但應當這樣做 **極為謹慎**特別是在高峰期,因為它可能導致 **資料損失** 或者說 **服務中斷**。 。 。 。

---

### 服務概覽

以下簡要描述在下文可見的每項服務。 **管理服務** 節:

| **服務名稱**         | **說明**                                                                                                                                         |
|--------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------|
| **客戶服務**       | 透過 SMPP 連線 iTextPRO 與您的 SMSC 連線用於管理 **短訊息 MT( 移動終止)** 財務報告和審定財務報表 **SMS MO( 移動原始碼)** 堵車                      |
| **DLR 服務**          | 實時處理 **交付報告** 並更新系統內的資訊狀態,以便準確報告。                                      |
| **SMPP 伺服器服務**  | 收聽從 **無害環境管理使用者** 上,允許外部使用者提交短訊息。                                      |
| **檔案拾取服務**  | 閱讀使用者上傳的運動訊息檔案並轉發到 **閘道器佇列** 用於處理。                                               |
| **驗證服務**    | 驗證 **約束性引數** 對於透過API和SMPP介面連線的使用者,確保只接受授權客戶端的流量.       |
| **資料構建服務** | 管理儲存 **PDU 日誌** 財務報告和審定財務報表 **訊息日誌** 輸入用於記錄和除錯目的的資料庫。                                          |
| **報表構建服務** | 簡介和彙編 **傳送的清點報告和簡要報告** 使用者,協助跟蹤使用和計費。                                            |
| **下載報告服務** | 程序和生成 **可下載報告** 根據請求的資料對管理員和使用者都適用。                                                  |
| **通知處** | 傳送 **電子郵件提醒通知** 給使用者和管理員,並積極監測 **閘道器健康狀況**。 。 。 。                                               |

---

### 關鍵特性

- 檢視 **現況** 每個服務。
- 備選 **開始** 或者說 **停下來** 來自網路介面的服務。
- 設計供 **高階使用者或管理員** 對服務的依賴性有深刻的理解。
- 每個服務包括元資料,如 **應用程式版本**。 。 。 。

---

### 最佳做法

- 在停止任何關鍵服務(如客戶端,DLR,驗證器)之前,總是確認高峰交通小時.
- 避免在沒有諮詢系統日誌或技術支援的情況下重新啟動服務。
- 確保資料庫和閘道器服務同步進行,以防止資料不一致。

---

這個 **管理服務** 特性提供了監測和控制系統級程序的集中方式,確保了iTextPRO環境中的平穩執行。
