
# 核准

这个 **核准** 一节使管理员能够审查和管理用户或转售商提出的所有批准请求。 这包括: **发件人标识** 财务报告和审定财务报表 **模板** 为审定和遵约目的提出的请求。

管理员可以审查提交的请求, **批准或拒绝** 视需要而定。 决定也可以是: **在任何时候重新审议**,允许驳回经批准的请求或核准被否决的请求。

---

## 所有发件人 ID 请求

本节显示 **所有发件人ID请求** 由用户或转售商发起,按状态分类:

- **待决**
- **核定数**
- **被拒绝**

管理员可以审查请求的细节,直接从本节采取适当行动。

![All Sender ID Requests](images/allsenderid1.png)

---

### 待发件人 ID 请求

在这里,你可以 **审查和采取行动** 正在等待用户和转售商的发件人ID批准请求。

**可用的行动 :**

- **核准**
- **拒绝**

---

### 核定发件人

此类列出全部 **已核准发件人ID**。 。 。 。

你有这个 **拒绝的灵活性** 提供对发送者ID生命周期管理的全面控制。

---

### 不允许的发件人ID

显示列表 **被拒绝的发件人ID**。 。 。 。

你们可以 **核准** 允许重新考虑和动态调整。

---

## 核准行动

管理员可以 **批准或拒绝** 任何发件人ID或模板请求。

- 一个 **经核准的请求可以驳回**,则a **驳回的请求可以得到批准**,如果需要重新考虑。
- 所有行动立即适用,并反映在请求状态中。

## 批量批准和拒绝

这个 **批量核准** 财务报告和审定财务报表 **批量拒绝** 选项允许管理员:

- 选择所有请求,或
- 选择特定请求

这个功能能够更快地在一次行动中处理多个请求.

---

!!! info "关键说明"
    - 核准控制措施适用于两者 **用户和转售商**。 。 。 。
    - 为了审计和跟踪目的,所有请求状况都可见。
    - 在整个申请中,批准状况的改变立即生效。
