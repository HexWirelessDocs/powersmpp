
# 核准

這個 **核准** 一節使管理員能夠審查和管理使用者或轉售商提出的所有批准請求。 這包括: **發件人標識** 財務報告和審定財務報表 **模板** 為審定和遵約目的提出的請求。

管理員可以審查提交的請求, **批准或拒絕** 視需要而定。 決定也可以是: **在任何時候重新審議**,允許駁回經批准的請求或核准被否決的請求。

---

## 所有發件人 ID 請求

本節顯示 **所有發件人ID請求** 由使用者或轉售商發起,按狀態分類:

- **待決**
- **核定數**
- **被拒絕**

管理員可以審查請求的細節,直接從本節採取適當行動。

![All Sender ID Requests](images/allsenderid1.png)

---

### 待發件人 ID 請求

在這裡,你可以 **審查和採取行動** 正在等待使用者和轉售商的發件人ID批准請求。

**可用的行動 :**

- **核准**
- **拒絕**

---

### 核定發件人

此類列出全部 **已核准發件人ID**。 。 。 。

你有這個 **拒絕的靈活性** 提供對傳送者ID生命週期管理的全面控制。

---

### 不允許的發件人ID

顯示列表 **被拒絕的發件人ID**。 。 。 。

你們可以 **核准** 允許重新考慮和動態調整。

---

## 核准行動

管理員可以 **批准或拒絕** 任何發件人ID或模板請求。

- 一個 **經核准的請求可以駁回**,則a **駁回的請求可以得到批准**,如果需要重新考慮。
- 所有行動立即適用,並反映在請求狀態中。

## 批次批准和拒絕

這個 **批次核准** 財務報告和審定財務報表 **批次拒絕** 選項允許管理員:

- 選擇所有請求,或
- 選擇特定請求

這個功能能夠更快地在一次行動中處理多個請求.

---

!!! info "關鍵說明"
    - 核准控制措施適用於兩者 **使用者和轉售商**。 。 。 。
    - 為了審計和跟蹤目的,所有請求狀況都可見。
    - 在整個申請中,批准狀況的改變立即生效。
