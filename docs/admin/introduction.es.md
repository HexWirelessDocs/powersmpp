# Preparación para pruebas en vivo

Facilitar **pruebas en vivo** en nuestro servidor, usted es requerido para compartir su **SMPP bind credenciales** para **SMS MO (Mobile Originated)** y **MT (Mobile Terminated)** conexiones. Estas credenciales se obtienen típicamente de su:

- **vendedor de SMS Gateway** o 
- **SMSC (Centro de servicio de mensajes cortos)**

Esta entidad juega un **función vital** en almacenamiento, reenvío, conversión y entrega de mensajes SMS a teléfonos móviles - típicamente representados por un **telecompañía** o a **tercero agregador**.

---

## Parámetros de conexión SMPP muestra

Tu típico **Telco o SMSC** le proporcionará los siguientes parámetros esenciales:

| Parámetro | Necesario | Descripción |
|-----------|:--------:|-------------|
| **Dirección IP de SMSC** | :material-check: | Dirección IP del servidor SMSC |
| **SMSC Port** | :material-check: | Número de puerto para conexión SMPP |
| **ID del sistema** | :material-check: | Nombre de usuario para la autenticación SMPP |
| **Contraseña** | :material-check: | Contraseña para autenticación SMPP |
| **Tipo de sistema** | | Identificador opcional para el sistema ESME |
| **Modo Bind** | :material-check: | Transmisor / receptor / transceptor |
| **Número de sesiones** | :material-check: | Tx / Rx / TxRx |
| **TON** | | Tipo de Número (por ejemplo, Internacional, Alphanumeric) |
| **NPI** | | Indicador del plan de numeración (por ejemplo, E.164, ISDN) |
