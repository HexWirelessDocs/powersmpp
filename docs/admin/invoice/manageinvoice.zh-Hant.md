
# 發票

發票模組允許管理員管理其直接使用者的賬單(賬戶型別: **使用者**透過提供一個外掛生成發票,下載發票,或透過電子郵件傳送發票. 發票生成流程是自動化的,由行政主管安排的具體間隔觸發。

這個模組幫助管理員有效管理其兒童使用者的計費業務。

---

## 作為外掛的發票[預付/後付]

行政官必須選擇 **發票執照** 分別。 一旦啟用,發票模組就可以在應用程式中使用選定的計費模式——**預付** 或者說 **離職償金**。 。 。 。

- 這個 **計費方法** 每個使用者的(預付/預付)是在使用者建立期間定義的.
- 管理員可以為每個使用者配置發票方法.
- 自動報名的使用者將擁有 **預付** 以違約方式計費。

---

## 發票板

當管理員訪問發票模組時,第一個檢視是 **發票板**儀表板提供了關鍵的收費見解:

### 1. 發票總額
顯示 **發票總數**,包括:
- 核定發票 
- 未核准的發票 

### 2. 支付發票
顯示數 **全額支付的發票**。 。 。 。

### 3. 未結算發票
顯示尚待處理的發票數量,包括:
- 未付發票 
- 部分付款的發票 

### 4. 管理發票
報告摘要 *管理發票* 節。 
管理員可以單擊 **跳轉到頁面** 導航到詳細的發票管理。

### 5. 管理經常性發票
適用 **僅供付薪使用者使用**。 。 。 。 
從這裡,管理員可以:
- 建立經常性發票 
- 開始或停止計費週期 
- 修改計費週期設定 

### 6. 管理收到的付款
報告摘要 *管理收到的付款* 節。 
管理員可以使用 **跳轉到頁面** 按鈕。

## 管理發票

這個 **管理發票** 頁面顯示所有發票(包括經常性發票)及其付款狀況。 管理員可以索取發票,下載發票,併發送付款提醒郵件.

![Manage Invoices](images/manage1.png)

### 關鍵特性 :

#### **下載發票報告**
允許管理員將發票報告匯出並下載為Excel工作表.

#### **傳送批次提醒**
允許向發票的使用者傳送付款提醒 **未支付** 或者說 **部分繳付**。 。 。 。

#### **草稿**
為郵資使用者自動生成的經常性發票作為 **草稿**。 。 。 。 
管理人員必須手動審查並核准行動選單中的草稿。

#### **高階搜尋**
高階搜尋選項允許透過下列方式過濾發票:
- 使用者 
- 發票身份證 
- 應有地位 
- 繳款情況 

這有助於行政部門根據具體要求迅速找到發票。

## 建立發票

本節允許管理員手動為使用者建立發票.

---

### **預付使用者:**

1. 選擇使用者型別 : **預付**
2. 選擇使用者 。
3. 輸入 **發票日期** 財務報告和審定財務報表 **到期日期**。 。 。 。
4. 輸入 **數額** (稅前.
5. 提供a **說明** 發票
6. 修改 **術語和條件** 或者說 **客戶說明** 如果需要(或使用預設配置的值)。
7. 選擇其中之一 :
   - **建立發票** (如果尚未收到付款)
   - **建立索賠發票( C)** (如果已經收到付款——該系統將迅速輸入付款細節)

---

### **對於付後用戶:**

事後支付的發票通常是在以下定義的計費週期結束時自動生成的: **經常性發票**。 。 。 。 
但是,如果一個週期被跳過或需要人工干預,管理員可以手動建立發票:

1. 選擇使用者型別 : **離職償金**
2. 選擇使用者 。
3. 選擇 **日期範圍** 應為其開具發票並取回記錄。
4. 該系統將顯示選定期間的通訊記錄。
5. 經核實後,選擇:
   - **建立發票**
   - **建立和索賠發票**
