
# 发票

发票模块允许管理员管理其直接用户的账单(账户类型: **用户**通过提供一个插件生成发票,下载发票,或通过电子邮件发送发票. 发票生成流程是自动化的,由行政主管安排的具体间隔触发。

这个模块帮助管理员有效管理其儿童用户的计费业务。

---

## 作为插件的发票[预付/后付]

行政官必须选择 **发票执照** 分别。 一旦启用,发票模块就可以在应用程序中使用选定的计费模式——**预付** 或者说 **离职偿金**。 。 。 。

- 这个 **计费方法** 每个用户的(预付/预付)是在用户创建期间定义的.
- 管理员可以为每个用户配置发票方法.
- 自动报名的用户将拥有 **预付** 以违约方式计费。

---

## 发票板

当管理员访问发票模块时,第一个视图是 **发票板**仪表板提供了关键的收费见解:

### 1. 发票总额
显示 **发票总数**,包括:
- 核定发票 
- 未核准的发票 

### 2. 支付发票
显示数 **全额支付的发票**。 。 。 。

### 3. 未结算发票
显示尚待处理的发票数量,包括:
- 未付发票 
- 部分付款的发票 

### 4. 管理发票
报告摘要 *管理发票* 节。 
管理员可以单击 **跳转到页面** 导航到详细的发票管理。

### 5. 管理经常性发票
适用 **仅供付薪用户使用**。 。 。 。 
从这里,管理员可以:
- 创建经常性发票 
- 开始或停止计费周期 
- 修改计费周期设置 

### 6. 管理收到的付款
报告摘要 *管理收到的付款* 节。 
管理员可以使用 **跳转到页面** 按钮。

---

## 管理发票

这个 **管理发票** 页面显示所有发票(包括经常性发票)及其付款状况。 管理员可以索取发票,下载发票,并发送付款提醒邮件.

![Manage Invoices](images/manage1.png)

### 关键特性 :

#### **下载发票报告**
允许管理员将发票报告导出并下载为Excel工作表.

#### **发送批量提醒**
允许向发票的用户发送付款提醒 **未支付** 或者说 **部分缴付**。 。 。 。

#### **草稿**
为邮资用户自动生成的经常性发票作为 **草稿**。 。 。 。 
管理人员必须手动审查并核准行动菜单中的草稿。

#### **高级搜索**
高级搜索选项允许通过下列方式过滤发票:
- 用户 
- 发票身份证 
- 应有地位 
- 缴款情况 

这有助于行政部门根据具体要求迅速找到发票。

## 创建发票

本节允许管理员手动为用户创建发票.

---

### **预付用户:**

1. 选择用户类型 : **预付**
2. 选择用户 。
3. 输入 **发票日期** 财务报告和审定财务报表 **到期日期**。 。 。 。
4. 输入 **数额** (税前.
5. 提供a **说明** 发票
6. 修改 **术语和条件** 或者说 **客户说明** 如果需要(或使用默认配置的值)。
7. 选择其中之一 :
   - **创建发票** (如果尚未收到付款)
   - **创建索赔发票( C)** (如果已经收到付款——该系统将迅速输入付款细节)

---

### **对于付后用户:**

事后支付的发票通常是在以下定义的计费周期结束时自动生成的: **经常性发票**。 。 。 。 
但是,如果一个周期被跳过或需要人工干预,管理员可以手动创建发票:

1. 选择用户类型 : **离职偿金**
2. 选择用户 。
3. 选择 **日期范围** 应为其开具发票并取回记录。
4. 该系统将显示选定期间的通讯记录。
5. 经核实后,选择:
   - **创建发票**
   - **创建和索赔发票**
