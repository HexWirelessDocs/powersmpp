
# 黑名單數字

這個 **黑名單數字** 特性允許管理員定義整個應用程式被遮蔽的移動數字列表. 一旦一個號碼被加入黑名單, **所有傳送到該號碼的資訊都會被應用程式自動拒絕**,則沒有使用者能夠向它傳送訊息。

## 目的

這一特徵用於防止向受限制的、無效的、不符合要求的或被禁號碼傳送資訊,以保持監管合規和控制濫用。

![Blacklist Numbers](images/blacklist2.png)

---

## 將數字新增到黑名單

1. 導航到 **黑名單數字** 區域。
2. 點選 **新增新內容**。 。 。 。
3. 輸入輸入欄位中的移動數字。
    - 透過用逗號將它們分開,可以同時新增多個數字(<span data-ph="0"></span>) (中文(簡體) ).
4. 確保 **每個手機號碼包括國家程式碼** (如:+91XXXXXXXXXX.
5. 儲存更改以應用整個應用程式的黑名單條目。

![Add Blacklist Numbers](images/blacklist3.png)

---

## 刪除列入黑名單的數字

應用程式支援 **批次刪除**,允許管理員在單一操作中刪除多個黑名單條目。

署長也可 **逐個刪除所選條目** 視需要而定。

![Bulk Delete Blacklist](images/blacklist4.png)

---

## 信件處理行為

!!! warning "被遮蔽的信件"
    - 傳送到黑名單號碼的任何信件 **在應用程式級別被遮蔽**。 。 。 。
    - 信件請求是 **立即駁回**,並且不會將其轉交供應商。

!!! tip
 確保每個移動號碼都包含 **國家程式碼** (如:+91XXXXXXXXXX),用於正確功能.
