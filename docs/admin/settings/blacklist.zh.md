
# 黑名单数字

这个 **黑名单数字** 特性允许管理员定义整个应用程序被屏蔽的移动数字列表. 一旦一个号码被加入黑名单, **所有发送到该号码的信息都会被应用程序自动拒绝**,则没有用户能够向它发送消息。

## 目的

这一特征用于防止向受限制的、无效的、不符合要求的或被禁号码发送信息,以保持监管合规和控制滥用。

![Blacklist Numbers](images/blacklist2.png)

---

## 将数字添加到黑名单

1. 导航到 **黑名单数字** 区域。
2. 点击 **添加新内容**。 。 。 。
3. 输入输入字段中的移动数字。
    - 通过用逗号将它们分开,可以同时添加多个数字(<span data-ph="0"></span>) (中文(简体) ).
4. 确保 **每个手机号码包括国家代码** (如:+91XXXXXXXXXX.
5. 保存更改以应用整个应用程序的黑名单条目。

![Add Blacklist Numbers](images/blacklist3.png)

---

## 删除列入黑名单的数字

应用程序支持 **批量删除**,允许管理员在单一操作中删除多个黑名单条目。

署长也可 **逐个删除所选条目** 视需要而定。

![Bulk Delete Blacklist](images/blacklist4.png)

---

## 信件处理行为

!!! warning "被屏蔽的信件"
    - 发送到黑名单号码的任何信件 **在应用程序级别被屏蔽**。 。 。 。
    - 信件请求是 **立即驳回**,并且不会将其转交供应商。

!!! tip
 确保每个移动号码都包含 **国家代码** (如:+91XXXXXXXXXX),用于正确功能.
