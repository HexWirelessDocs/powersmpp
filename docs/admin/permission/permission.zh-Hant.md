

# 許可權

## iTextPRO 中的賬戶角色執行許可權

單位 **電子文字PRO**,管理員和轉售商可給予特定特權 **賬戶角色管理員**,使其進入指定的工作區。 
可以建立多重行政作用,以履行各種職責。

### 示例角色

**財務小組的作用**
- QQ 賬單部分訪問
- 報告科
- 其他相關特權

**NOC 團隊角色**
- QQ 監視日誌訪問
- 報告科
- 管理選項
- 其他相關特權

**登入程式 :**
- **行政主管:** 使用管理員的登入 URL 和 Port 。
- **Reseller的行政部門:** 使用 Reseller 的登入 URL 和 port 。

這種隔離確保高管有安全和針對具體角色的登入經驗,簡化了與其職責相符的功能。

## 許可權類別

對iTextPRO中執行使用者的許可根據訪問級別和模組內的行動進行分類.

### 類別

**檢視** 
- 可以在指定的模組內檢視內容 。 
- *不允許修改或刪除。*

**新增** 
- 可以在模組中新增新條目 。 
- *無法編輯或刪除已存在的條目 。*

**編輯** 
- 可編輯/更新已有條目 。 
- *無刪除許可權 。*

**刪除** 
- 可以刪除模組中的條目 。 
- *沒有編輯許可權 。*

**全控** 
- 授予特定模組的檢視、新增、編輯和刪除許可權。 
- 無所限入.

## 全控等級( S)

單位 **電子文字PRO**, (中文(簡體) ). **全控** 提供全面准入。

**活性假設:**
1. 啟用全控 **頂級** 自動授予所有子模組.
2. 讓它成為一個 **特定模組** 覆蓋檢視、新增、編輯和刪除。

**等級 :**
- 級別代表不同的科,功能,或資料類別.
- 上一級級聯的全能控制,所有分級.

**明確殘疾:**
- 子模組中的特定許可權可以手動被停用,即使是全控.

## 許可權型別

### 個人使用者許可權
- 授予特定執行使用者許可權.

**程式:**
1. 轉到 **管理許可權** 頁 次
2. 選擇 **個人** 作為許可型別。
3. 從下拉中選擇一個使用者 。
4. 點選 **顯示** 用於檢視/管理許可權。

![Individual Permissions](images/permission4.png)

---

### 執行使用者組許可權
- 授予一個團體的許可權;由所有團體成員繼承.

**程式:**
1. 轉到 **管理執行使用者組** 頁 次
2. 點選 **新增使用者組**。 。 。 。
3. 輸入組名稱並選擇使用者 。
4. 救護群生.
5. 使用 **使用者共計** 以檢視成員。
6. 根據需要編輯或刪除組。

📌 *如果一個執行官同時擁有使用者專用許可權和組許可權,他們就同時獲得.*

## 模組可見度

- 如果執行官沒有許可權 他們就會看到 **"拒絕進入"** 頁面。
- 如果在登入時取消許可權, 動作將顯示 **"拒絕進入"**。 。 。 。
- 模組根據所授予的許可權依次顯示。

示例 
- 若為 **配置** 財務報告和審定財務報表 **計費** 僅提供這些模組。
