

# 权限

## iTextPRO 中的账户角色执行权限

单位 **电子文本PRO**,管理员和转售商可给予特定特权 **账户角色管理员**,使其进入指定的工作区。 
可以建立多重行政作用,以履行各种职责。

### 示例角色

**财务小组的作用**
- QQ 账单部分访问
- 报告科
- 其他相关特权

**NOC 团队角色**
- QQ 监视日志访问
- 报告科
- 管理选项
- 其他相关特权

**登录程序 :**
- **行政主管:** 使用管理员的登录 URL 和 Port 。
- **Reseller的行政部门:** 使用 Reseller 的登录 URL 和 port 。

这种隔离确保高管有安全和针对具体角色的登录经验,简化了与其职责相符的功能。

## 权限类别

对iTextPRO中执行用户的许可根据访问级别和模块内的行动进行分类.

### 类别

**视图** 
- 可以在指定的模块内查看内容 。 
- *不允许修改或删除。*

**添加** 
- 可以在模块中添加新条目 。 
- *无法编辑或删除已存在的条目 。*

**编辑** 
- 可编辑/更新已有条目 。 
- *无删除权限 。*

**删除** 
- 可以删除模块中的条目 。 
- *没有编辑权限 。*

**全控** 
- 授予特定模块的查看、添加、编辑和删除权限。 
- 无所限入.

## 全控等级( S)

单位 **电子文本PRO**, (中文(简体) ). **全控** 提供全面准入。

**活性假设:**
1. 启用全控 **顶级** 自动授予所有子模块.
2. 让它成为一个 **特定模块** 覆盖视图、添加、编辑和删除。

**等级 :**
- 级别代表不同的科,功能,或数据类别.
- 上一级级联的全能控制,所有分级.

**明确残疾:**
- 子模块中的特定权限可以手动被禁用,即使是全控.

## 权限类型

### 个人用户权限
- 授予特定执行用户权限.

**程序:**
1. 转到 **管理权限** 页 次
2. 选择 **个人** 作为许可类型。
3. 从下拉中选择一个用户 。
4. 点击 **显示** 用于查看/管理权限。

![Individual Permissions](images/permission4.png)

---

### 执行用户组权限
- 授予一个团体的权限;由所有团体成员继承.

**程序:**
1. 转到 **管理执行用户组** 页 次
2. 点击 **添加用户组**。 。 。 。
3. 输入组名称并选择用户 。
4. 救护群生.
5. 使用 **用户共计** 以查看成员。
6. 根据需要编辑或删除组。

📌 *如果一个执行官同时拥有用户专用权限和组权限,他们就同时获得.*

![Group Permissions](images/permission5.png)

## 模块可见度

- 如果执行官没有权限 他们就会看到 **"拒绝进入"** 页面。
- 如果在登录时取消权限, 动作将显示 **"拒绝进入"**。 。 。 。
- 模块根据所授予的权限依次显示。

示例 
- 若为 **配置** 财务报告和审定财务报表 **计费** 仅提供这些模块。
