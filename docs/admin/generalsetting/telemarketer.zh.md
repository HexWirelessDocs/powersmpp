
# 电话营销器配置

引进 **TRAI DLT 条例**印度的短信流量必须遵循强制性 **PE-TM链** (主要实体-电信商链) 这一链条确保了参与信息发送的所有利益攸关方的可追踪性和监管合规性,包括: **用户、Reseller、应用程序和供应商**。 。 。 。

为了遵守这些条例,应用程序必须配置为: **附加所需的Telemarketer信息** 在向上游供应商或短信中心提交信息之前。

---

## PE-TM 链式形成逻辑

PE-TM链在提交信件时根据 **供应商类型**。 。 。 。

### 信件流动

1. 这个 **用户** 随信件一起提交 **主要实体标识(PEID)**。 。 。 。
2. 这个 **应用程序** 附加其配置 **电话推销员身份证(TMID)**。 。 。 。
3. 连锁店的最终结构取决于供应商类型.

---

## 供应商类型和链条形成

{\fn华文仿宋\fs16\1cHD1D1D1}“隔离者”

 一个 **聚合器** 是一个中间供应商,向另一个供应商或短信中心进一步转发信息。

    - 应用程序附加其 **调试器**
    - 集合器附加它自己的 **TMID2 设备**
    - 将信件转发到下一跳

 **链成型 :** <span data-ph="0"></span>

    !!! info
 哈兴是 **无需** 在综合供应商申请一级。

“交付伙伴”

 A 类 **交付伙伴** 是负责将消息传送给听筒的最后短信。

    - 应用程序附加其 **调试器**
    - 最后一个环节必须是 **散开** 提交前

 **链成型 :** <span data-ph="0"></span>

    !!! info
 散列车确保遵守TRAI的安全和诚信要求.

## 配置步骤

1. 点击 **添加新内容**
2. 选择 **网关名称**
3. 输入 **电话推销员身份证(TMID)**
4. 根据供应商类型配置散列 :
    - **聚合器 :** 设定 **正在运行中 = 关闭**
    - **执行伙伴:** 设定 **正在运行中 = O**
5. **散列类型 :**
    - 默认为 **SHA-256 (英语).** (建议和TRAI遵守)
6. 点击 **保存** 以应用配置

![Telemarketer Configuration Dialog](images/telemarketer2.png)

---

!!! danger "重要说明"
    - 不正确的散列配置可能导致 **DLT 拒绝** 由供应商或短信中心提供。
    - 确保供应商类型(Agreator vs 交付伙伴)在配置前得到确认.
    - SHA-256作为默认散列算法被执行,以达到监管标准.
