
# 電話營銷器配置

引進 **TRAI DLT 條例**印度的簡訊流量必須遵循強制性 **PE-TM鏈** (主要實體-電信商鏈) 這一鏈條確保了參與資訊傳送的所有利益攸關方的可追蹤性和監管合規性,包括: **使用者、Reseller、應用程式和供應商**。 。 。 。

為了遵守這些條例,應用程式必須配置為: **附加所需的Telemarketer資訊** 在向上遊供應商或簡訊中心提交資訊之前。

---

## PE-TM 鏈式形成邏輯

PE-TM鏈在提交信件時根據 **供應商型別**。 。 。 。

### 信件流動

1. 這個 **使用者** 隨信件一起提交 **主要實體標識(PEID)**。 。 。 。
2. 這個 **應用程式** 附加其配置 **電話推銷員身份證(TMID)**。 。 。 。
3. 連鎖店的最終結構取決於供應商型別.

---

## 供應商型別和鏈條形成

{\fn華文仿宋\fs16\1cHD1D1D1}“隔離者”

 一個 **聚合器** 是一箇中間供應商,向另一個供應商或簡訊中心進一步轉發資訊。

    - 應用程式附加其 **偵錯程式**
    - 集合器附加它自己的 **TMID2 裝置**
    - 將信件轉發到下一跳

 **鏈成型 :** <span data-ph="0"></span>

    !!! info
 哈興是 **無需** 在綜合供應商申請一級。

“交付夥伴”

 A 類 **交付夥伴** 是負責將訊息傳送給聽筒的最後簡訊。

    - 應用程式附加其 **偵錯程式**
    - 最後一個環節必須是 **散開** 提交前

 **鏈成型 :** <span data-ph="0"></span>

    !!! info
 雜湊車確保遵守TRAI的安全和誠信要求.

## 配置步驟

1. 點選 **新增新內容**
2. 選擇 **閘道器名稱**
3. 輸入 **電話推銷員身份證(TMID)**
4. 根據供應商型別配置雜湊 :
    - **聚合器 :** 設定 **正在執行中 = 關閉**
    - **執行夥伴:** 設定 **正在執行中 = O**
5. **雜湊型別 :**
    - 預設為 **SHA-256 (英語).** (建議和TRAI遵守)
6. 點選 **儲存** 以應用配置

![Telemarketer Configuration Dialog](images/telemarketer2.png)

---

!!! danger "重要說明"
    - 不正確的雜湊配置可能導致 **DLT 拒絕** 由供應商或簡訊中心提供。
    - 確保供應商型別(Agreator vs 交付夥伴)在配置前得到確認.
    - SHA-256作為預設雜湊演算法被執行,以達到監管標準.
