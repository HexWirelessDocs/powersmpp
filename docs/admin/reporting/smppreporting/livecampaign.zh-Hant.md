
# 現場活動報告

這個 **現場活動報告** 模組使管理人員能夠實時瞭解整個平臺發起的所有活動。

雖然終端使用者只能從使用者面板中檢視自己的活動,但本節為管理員提供了對建立的活動的集中檢視。 **任何使用者賬戶** 在系統內。

這個模組旨在幫助管理員:

- 監測積極和已完成的運動
- 實時跟蹤活動處理情況
- 按使用者或日期過濾工作
- 必要時干預和停止運動
- 分析交付狀況分佈

## 獲取現場運動報告

獲取現場活動報告:

1. 登入管理小組。
2. 導航到 **報告**。 。 。 。
3. 選擇 **現場活動報告**。 。 。 。

啟用後,系統將顯示在預設日期範圍內(通常是今天)發起的活動清單。

## 過濾選項

該模組提供了靈活的過濾選項來幫助管理員快速定位特定運動.

### A. 使用者過濾器

管理員可以:

- 從中選擇一個特定的使用者賬戶 **使用者過濾器**
- 只檢視該選定使用者發起的活動

這在下列情況下是有用的:

- 調查支助問題
- 監測大量傳送者
- 審計活動

### B. 選擇日期範圍

運動可以使用 **首選範圍** 選項 :

- **現在** - 顯示當前發起的活動
- **昨日來** - 顯示上一個日期發起的活動
- **自定義範圍** - 允許選擇具體的開始和結束日期

要使用自定義區域 :

1. 選擇“ 自定義 ”
2. 選擇想要的開始日期
3. 選擇結束日期
4. 應用過濾器

該系統將獲取在選定時限內啟動的所有活動。

![User Filter and Date Range Selection](images/livecampaign2.png)

## 運動監測

如果在任何使用者賬戶下執行一個大型活動,管理員可以實時監控其處理.

透過點選 **顯示狀態** 介面中可用的按鈕 :

- 管理員可以檢視當前訊息處理細節.

這一特點對於需要密切監測的大量宣傳活動特別有用。

![Campaign Status Dialog](images/livecampaign3.png)

---

## 停止運動

如果需要,管理員有能力停止正在進行的競選活動。

這個 **停下來** 按鈕 **行動** 欄。

!!! danger "停止運動"
    - 點選 **停下來** 按鈕。
    - 這項運動將停止進一步進行。
    - 已經處理的郵件可能不會受到影響,視其階段而定.

---

## 動作按鈕 — status-Wise 計數

這個 **行動** 按鈕還允許供應商或管理員檢查每個作業的詳細狀態計數。

透過使用此選項,管理員可以:

- 在每個傳送狀態下檢視信件的計數 。
- 分析競選表現。
- 查明任何異常故障或待決模式。

透過點選它,管理員可以檢視完整的狀態計數信件,包括:

- **已提交** 計數
- **已交付** 計數
- **未交付** 計數
- **被拒絕** 計數
- **過期** 計數(如果適用)
- **未知** 計數
- **其他人員** 計數

![Status-Wise Message Count](images/livecampaign4.png)
