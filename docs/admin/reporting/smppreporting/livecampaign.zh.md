
# 现场活动报告

这个 **现场活动报告** 模块使管理人员能够实时了解整个平台发起的所有活动。

虽然终端用户只能从用户面板中查看自己的活动,但本节为管理员提供了对创建的活动的集中视图。 **任何用户账户** 在系统内。

这个模块旨在帮助管理员:

- 监测积极和已完成的运动
- 实时跟踪活动处理情况
- 按用户或日期过滤工作
- 必要时干预和停止运动
- 分析交付状况分布

## 获取现场运动报告

获取现场活动报告:

1. 登录管理小组。
2. 导航到 **报告**。 。 。 。
3. 选择 **现场活动报告**。 。 。 。

启用后,系统将显示在默认日期范围内(通常是今天)发起的活动清单。

![Live Campaign Report Overview](images/livecampaign1.png)

## 过滤选项

该模块提供了灵活的过滤选项来帮助管理员快速定位特定运动.

### A. 用户过滤器

管理员可以:

- 从中选择一个特定的用户账户 **用户过滤器**
- 只查看该选定用户发起的活动

这在下列情况下是有用的:

- 调查支助问题
- 监测大量发送者
- 审计活动

### B. 选择日期范围

运动可以使用 **首选范围** 选项 :

- **现在** - 显示当前发起的活动
- **昨日来** - 显示上一个日期发起的活动
- **自定义范围** - 允许选择具体的开始和结束日期

要使用自定义区域 :

1. 选择“ 自定义 ”
2. 选择想要的开始日期
3. 选择结束日期
4. 应用过滤器

该系统将获取在选定时限内启动的所有活动。

![User Filter and Date Range Selection](images/livecampaign2.png)

## 运动监测

如果在任何用户账户下运行一个大型活动,管理员可以实时监控其处理.

通过点击 **显示状态** 界面中可用的按钮 :

- 管理员可以查看当前消息处理细节.

这一特点对于需要密切监测的大量宣传活动特别有用。

![Campaign Status Dialog](images/livecampaign3.png)

---

## 停止运动

如果需要,管理员有能力停止正在进行的竞选活动。

这个 **停下来** 按钮 **行动** 栏。

!!! danger "停止运动"
    - 点击 **停下来** 按钮。
    - 这项运动将停止进一步进行。
    - 已经处理的邮件可能不会受到影响,视其阶段而定.

---

## 动作按钮 — status-Wise 计数

这个 **行动** 按钮还允许供应商或管理员检查每个作业的详细状态计数。

通过使用此选项,管理员可以:

- 在每个发送状态下查看信件的计数 。
- 分析竞选表现。
- 查明任何异常故障或待决模式。

通过点击它,管理员可以查看完整的状态计数信件,包括:

- **已提交** 计数
- **已交付** 计数
- **未交付** 计数
- **被拒绝** 计数
- **过期** 计数(如果适用)
- **未知** 计数
- **其他人员** 计数

![Status-Wise Message Count](images/livecampaign4.png)
