
# 閘道器佇列

![Gateway Queue Overview](images/gatewayqueue1.png) 
![Gateway Queue Details](images/gatewayqueue2.png)

單位 **電子文字PRO**,有效管理簡訊流量是一個優先事項。 
以下是平臺如何處理簡訊流量,並提供基本的見解:

## 交通積累和排隊管理
- iTextPRO從各種使用者介面收集簡訊流量.
- 累計流量被排隊提交供應商閘道器。
- 使用者可以檢查 **佇列數** 為每個供應商閘道器測量待處理流量。
- 佇列資訊為 **動態** ,並可以重新整理以顯示最近數。

## 可見於供應商閘道器會話
- 向供應商閘道器的連線和活躍會議提供能見度。
- 使用者可以監視 **活動連線** 在實時。
- 會話區域中沒有活動連線可指示 a **網路斷電**,表示潛在的連通性問題.

## 動態資訊更新
- 所有與簡訊流量,佇列計數,閘道器會話相關的資訊都是動態的.
- 使用者可以 **重新整理資料** 獲取最新和最準確的資訊。
- 實時更新可確保使用者目前對簡訊流量和閘道器連線狀況有深入瞭解.

## 網路故障警報
- 屆會領域缺乏積極聯絡可構成 **網路斷電警告**。 。 。 。
- iTextPRO為潛在的連線問題提供了能見度,使使用者能夠及時採取行動.

---

iTextPRO 軟體 **SMS 交通管理** 功能旨在向用戶提供 **實時資訊**能夠進行有效的監測 **積極措施** 與網路有關的問題。
